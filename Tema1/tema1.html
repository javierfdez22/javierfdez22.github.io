Pila de Ejercicios 2

  1. Define "Ciclo de vida del software".

    2. Son todas las fases que tenemos que llevar a cabo para desarrollar y crear un software, desde su análisis para poder empezar a desarrollarlo hasta su mantenimiento una vez lo tenemos hecho.


  2. Nombra las fases principales del desarrollo de software y explica brevemente que se hace en cada una de ellas.
    • Análisis: Se definen las necesidades del cliente, las cuales el software debe cumplir.
    • Diseño: Se descompone y organiza el sistema en elementos componentes que pueden ser desarrollados por separado.
    • Codificación: Se escribe el código fuente de cada componente.
    • Pruebas: Se somete el programa al mayor número de situaciones diferentes para encontrar fallos.
    • Mantenimiento: Consiste en realizar cambios ocasionales una vez tenemos creado el software, para corregir errores, mejorar una funcionalidad...

  3. Explica brevemente en qué consiste el modelo en cascada cuando hablamos de desarrollo de software.

    4. Es un modelo de desarrollo bastante estricto, ya que para poder pasar a la siguiente fase es necesario acabar con la anterior, por lo tanto, no podemos ver un resultado hasta finalizar todas las fases.


  4. Ventajas e inconvenientes del modelo en cascada.

    5. Ventajas:

    • Estructura sencilla.
    • Buena documentación del proceso.
    • Se puede representar cronológicamente de forma sencilla.

    Inconvenientes:

    • El cliente no ve el producto hasta su finalización.
    • Se adapta mal al cambio continuo de especificaciones.
    • Hay diferentes variantes con un mayor o menor número de actividades.

  5. ¿Qué se entiende por verificación? ¿Y por validación?

    6. La verificación es una evaluación del sistema o componentes para determinar si los productos de una fase de desarrollo satisfacen las condiciones impuestas en el inicio de una etapa.
    La validación también es una evaluación del sistema o de componentes, solamente que se da en el transcurso o al final del proceso del desarrollo, donde se determina si cumple con lo especificado.

    El equipo de desarrollo o Scrum Team, se encarga de desarrollar Iteraciones o Sprint regulares (cada 2, 3 o 4 semanas), de esta manera, cada poco tiempo podemos obtener resultados parciales que entregar al cliente.
